# พรอมต์อัปเกรดระบบสอบออนไลน์ (Backend & AI Logic)

## 1. พรอมต์ยกระดับสมการ AI ใบหน้า (Advanced 3D Head Pose & Liveness)
**เป้าหมาย:** เพิ่มความแม่นยำด้วยคณิตศาสตร์ 3 มิติ และตรวจจับคนเป็นๆ (Liveness Detection)

> ทำหน้าที่เป็น AI Computer Vision Engineer ฉันมีระบบตรวจจับใบหน้าด้วย MediaPipe อยู่แล้ว (รับข้อมูลมาเป็นจุด Landmarks) ฉันต้องการอัปเกรดลอจิกการคำนวณเบื้องหลัง **โดยมีกฎเหล็กคือห้ามแก้ไขส่วนแสดงผลหรือ UI เด็ดขาด ให้คืนค่ากลับมาเป็น Data เท่านั้น**:
> 1. เปลี่ยนวิธีการคำนวณการหันหน้า (Head Pose) จากเดิมที่ใช้อัตราส่วนแกน X, Y ให้ไปใช้เทคนิค **SolvePnP** (Perspective-n-Point) เพื่อคำนวณองศา Pitch, Yaw, Roll แบบ 3D ที่แม่นยำระดับองศา
> 2. เพิ่มลอจิกคำนวณ **Liveness Detection** โดยใช้สมการ Eye Aspect Ratio (EAR) ร่วมกับ Mouth Aspect Ratio (MAR) เพื่อสร้างตัวแปรตรวจสอบว่าผู้สอบกำลังกะพริบตาหรือหายใจตามปกติหรือไม่ ป้องกันการนำรูปถ่ายมาบังกล้อง
> ขอตัวอย่างโค้ดลอจิกคณิตศาสตร์นี้ พร้อมคอมเมนต์อธิบายสมการอย่างละเอียด

**💡 อธิบายการทำงาน:**
* **SolvePnP:** เปลี่ยนจากการกะระยะบนภาพ 2 มิติ เป็นการสร้าง "โมเดล 3 มิติ" ของหน้าผู้สอบ ทำให้ระบบรู้ว่าก้ม เงย หรือหันซ้ายขวากี่องศาแบบเป๊ะๆ 
* **Liveness Detection:** คำนวณระยะห่างของเปลือกตาและริมฝีปากแบบเรียลไทม์ เพื่อดูว่ามีการกะพริบตาหรือขยับปากตามธรรมชาติหรือไม่ ถ้านิ่งผิดปกติ AI จะแยกออกทันทีว่าเป็นเพียงรูปถ่าย

## 2. พรอมต์สถาปัตยกรรม Backend รับโหลดมหาศาล (High-Concurrency Server)
**เป้าหมาย:** ออกแบบเซิร์ฟเวอร์ให้ทนทาน รับผู้สอบจำนวนมากได้โดยไม่ล่ม และสกัดกั้นการยิงสแปม

> ทำหน้าที่เป็น Backend Cloud Architect ฉันต้องการสร้าง Backend API และ WebSocket (เลือกใช้ Node.js หรือ Python) เพื่อรับข้อมูล Event การทุจริตจากฝั่ง Client **โดยไม่ต้องเขียนโค้ดฝั่งหน้าเว็บ**:
> 1. ออกแบบ Data Pipeline ที่สามารถรับ Request พร้อมๆ กันจากนักเรียน 500-1000 คนโดยที่เซิร์ฟเวอร์ไม่ล่ม ขอคำแนะนำและโครงสร้างโค้ดในการใช้ **Message Queue (เช่น Redis หรือ RabbitMQ)** เข้ามาช่วยพักข้อมูลก่อนเขียนลง Database
> 2. เขียนลอจิก **Rate Limiting และ Data Validation** เพื่อตรวจสอบแพ็กเกจข้อมูลที่รับเข้ามา ป้องกันนักเรียนสายไอทีเขียนสคริปต์ยิง API ขยะ (Spam) ใส่เซิร์ฟเวอร์เพื่อป่วนระบบ
> ขอโครงสร้างโค้ด Backend ที่เน้นความปลอดภัยและประสิทธิภาพสูงสุด

**💡 อธิบายการทำงาน:**
* **Message Queue:** ทำหน้าที่เหมือน "ระบบบัตรคิว" เมื่อมีข้อมูลหลั่งไหลเข้ามาพร้อมกันจำนวนมาก ระบบจะนำข้อมูลมารอในคิวก่อน แล้วทยอยบันทึกลงฐานข้อมูลอย่างเป็นระเบียบ ป้องกันฐานข้อมูลค้างและเซิร์ฟเวอร์ล่ม
* **Rate Limiting & Validation:** ระบบด่านตรวจที่คอยบล็อกการส่งข้อมูลที่ถี่เกินปกติ (เช่น ส่งข้อมูล 100 ครั้งใน 1 วินาที) หรือข้อมูลที่ถูกดัดแปลง (Script) เพื่อป้องกันการแฮกหลังบ้าน

## 3. พรอมต์สร้างอัลกอริทึมจับผิดด้วยสถิติ (Statistical Anomaly Detection)
**เป้าหมาย:** ใช้หลักการทางสถิติและพฤติกรรมภาพรวมเพื่อจับผิดผู้สอบที่เนียนโกง

> ทำหน้าที่เป็น Data Scientist และ Backend Developer ฉันมีฐานข้อมูลที่เก็บ Log การทำข้อสอบและสถิติ AI ของผู้สอบทุกคน ช่วยออกแบบและเขียนลอจิกอัลกอริทึมบน Backend ฝั่งเซิร์ฟเวอร์ เพื่อทำ **Behavioral Anomaly Detection (การวิเคราะห์พฤติกรรมผิดปกติ)**:
> 1. เขียนฟังก์ชันคำนวณค่าทางสถิติ เช่น หาค่าเฉลี่ย (Mean) และค่าเบี่ยงเบนมาตรฐาน (Standard Deviation) ของเวลาที่ใช้ทำข้อสอบแต่ละข้อของเด็กทั้งห้อง (Baseline)
> 2. หากนักเรียนคนใดมีแพทเทิร์นการตอบที่หลุดจากค่าเบี่ยงเบนมาตรฐาน (เช่น ทำเสร็จเร็วกว่าเพื่อนทั้งห้องแบบผิดปกติ) หรือมีค่าความถี่ในการมองซ้าย/ขวา สูงกว่าค่าเฉลี่ยของห้องเกิน 3-Sigma ให้ Backend คำนวณ 'คะแนนความน่าสงสัยสะสม' เพิ่มเข้าไปเอง
> ขอโค้ดลอจิกคณิตศาสตร์นี้ (Node.js หรือ Python) โดยเน้นไปที่อัลกอริทึมการคำนวณล้วนๆ

**💡 อธิบายการทำงาน:**
* **Anomaly Detection:** นำข้อมูลพฤติกรรมการสอบของนักเรียนทั้งห้องมาคำนวณหา "ค่าเฉลี่ยมาตรฐาน" (Baseline) หากมีผู้สอบคนใดทำพฤติกรรมที่หลุดออกจากกลุ่มอย่างชัดเจน (เช่น กาข้อสอบเร็วผิดมนุษย์ หรือขยับหน้าบ่อยจนเกินค่าเฉลี่ย) ระบบจะบวกคะแนนความน่าสงสัยให้ทันที เป็นการใช้สถิติจับโป๊ะที่หลบเลี่ยงได้ยาก
