# PROCTOR.sys — AI Proctoring System\n\n🎯 **Advanced exam proctoring with real-time AI detection**\n\nVersion 2.0 with optimized performance and enhanced detection accuracy.\n\n## Features ✨\n\n- **Real-time Gaze Tracking** - Detects when students look away from screen\n- **Head Pose Detection** - Monitors head rotation and unusual postures\n- **Face Recognition** - Alerts on multiple faces in frame\n- **Suspicious Object Detection** - Identifies phones, books, laptops, tablets, etc.\n- **Evidence Capture** - Screenshots of suspicious activities with timestamps\n- **Export Logs** - CSV export for investigation and audit trails\n- **Smooth 3D Calibration** - Advanced head pose compensation\n- **Low Latency** - Optimized frame skipping for 30+ FPS\n\n## System Requirements\n\n- Modern browser with WebRTC support (Chrome, Edge, Firefox)\n- Webcam with minimum 640x480 resolution\n- HTTPS or localhost environment (required for camera access)\n- Minimum 4GB RAM\n- Stable internet connection (for MediaPipe models)\n\n## Installation\n\n### Backend (Python - Optional)\n\n```bash\n# Install dependencies\npip install opencv-python mediapipe ultralytics\n\n# For NVIDIA GPU support (optional)\npip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118\n\n# Run Python proctoring script\npython proctoring_optimized.py\n```\n\n### Frontend Setup\n\n```bash\n# Install Node.js dependencies\nnpm install\n\n# Development server\nnpm run dev\n\n# Production build\nnpm run build\n\n# Preview production build\nnpm run preview\n```\n\n## Configuration\n\nEdit `config.js` to customize thresholds:\n\n```javascript\nimport { CONFIG } from './config.js';\n\n// Adjust detection sensitivity\nCONFIG.DETECTION.GAZE_THRESHOLD = 10;  // Higher = less sensitive\nCONFIG.DETECTION.NOFACE_THRESHOLD = 15;\n```\n\n## Usage\n\n1. **Start the application**\n   ```bash\n   npm run dev\n   ```\n\n2. **Open browser** - Navigate to `http://localhost:5173`\n\n3. **Grant camera permissions** - Click allow when prompted\n\n4. **Start proctoring** - Click \"▶ เริ่มตรวจสอบ\" button\n\n5. **Monitor alerts** - Check right panel for real-time alerts\n\n6. **Export logs** - Click \"↓ Export CSV\" for audit trail\n\n## Alert Types\n\n| Type | Severity | Description |\n|------|----------|-------------|\n| 👁️ Gaze | Warning | Student looking away continuously |\n| 🔄 Head | Warning | Unusual head rotation/posture |\n| 😶 No Face | Danger | Student not in frame |\n| 👥 Multi-Face | Danger | Multiple people detected |\n| 📱 Object | Danger | Suspicious items detected |\n\n## Performance Optimization\n\n### Frame Skipping Strategy\n\n- **Face Mesh**: Every frame (smooth tracking)\n- **Face Detection**: Every 3 frames (detect multiple faces)\n- **Object Detection**: Every 5 frames (COCO-SSD)\n- **Result Caching**: Uses latest results between detections\n\n### Detection Thresholds\n\n```javascript\nGAZE_THRESHOLD: 8        // Alert after 8 consecutive off-center frames\nNOFACE_THRESHOLD: 12     // Alert after 12 frames without face\nCOLEDOWN: 4000ms         // Wait 4s before next alert of same type\n```\n\n## Calibration Guide\n\n### Fine-tuning Gaze Detection\n\n1. Open browser console (F12)\n2. Check gaze direction output\n3. Adjust thresholds in `detection.js`:\n   - `HORIZONTAL_THRESHOLD_LEFT` (0.35)\n   - `HORIZONTAL_THRESHOLD_RIGHT` (0.65)\n   - `VERTICAL_THRESHOLD_UP` (0.30)\n   - `VERTICAL_THRESHOLD_DOWN` (0.70)\n\n### Camera Positioning\n\n- **Optimal**: Eye level, 60-80cm away\n- **Lighting**: Soft, even lighting; avoid backlighting\n- **Resolution**: 640x480 minimum; 1280x720 recommended\n- **FPS**: 24-30 FPS ideal for smooth tracking\n\n## Architecture\n\n```\n┌─────────────────────┐\n│   Video Stream      │ (WebRTC)\n└──────────┬──────────┘\n           │\n           ├─→ [MediaPipe FaceMesh]     (Face landmarks, gaze)\n           ├─→ [MediaPipe FaceDetection] (Multiple face detection)\n           └─→ [TensorFlow COCO-SSD]   (Object detection)\n                     │\n                     ├─→ [Alert Engine]  (Thresholding, cooldown)\n                     │\n                     └─→ [UI Renderer]   (Real-time display)\n```\n\n## Troubleshooting\n\n### Camera Not Working\n\n1. **Check browser console** for specific error\n2. **HTTPS required** - Use localhost or HTTPS only\n3. **Camera in use** - Close Zoom/Teams/other apps\n4. **Permissions** - Check browser settings for camera access\n\n### Poor Detection Accuracy\n\n- Increase `minDetectionConfidence` in `App.jsx`\n- Improve lighting conditions\n- Adjust camera distance and angle\n- Increase gaze/head thresholds for stability\n\n### High CPU Usage\n\n- Increase `FRAME_SKIP_OBJECT_DETECTION` (currently 5)\n- Reduce video resolution (640x480 vs 1280x720)\n- Disable object detection if not needed\n- Use Python backend with GPU acceleration\n\n## Browser Compatibility\n\n| Browser | Support | Notes |\n|---------|---------|-------|\n| Chrome  | ✅ Full | Recommended |\n| Edge    | ✅ Full | Chromium-based |\n| Firefox | ✅ Full | Works well |\n| Safari  | ⚠️ Partial | WebRTC support may vary |\n| IE 11   | ❌ None | Not supported |\n\n## API Reference\n\n### getGazeDirection(landmarks)\nReturns: `'center' | 'left' | 'right' | 'up' | 'down'`\n\n### getHeadDirection(landmarks)\nReturns: `'center' | 'left' | 'right' | 'up' | 'down'`\n\n### drawOverlays(canvas, video, landmarks, headDir, suspiciousObjects, extraFaces)\nDraws real-time visualization on canvas\n\n## Data Privacy\n\n- **Local Processing** -
