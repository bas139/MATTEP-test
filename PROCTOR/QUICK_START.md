# 🚀 Quick Start Guide - PROCTOR v2.0\n\n## 5-Minute Setup\n\n### Step 1: Install Dependencies\n\n```bash\ncd c:\\Users\\Depa\\Downloads\\camay\\camay\nnpm install\n```\n\n### Step 2: Start Development Server\n\n```bash\nnpm run dev\n```\n\n**Output**:\n```\n➜  Local:   http://localhost:5173\n```\n\n### Step 3: Open in Browser\n\n- Click the link or navigate to `http://localhost:5173`\n- Grant camera permissions when prompted\n- Click \"▶ เริ่มตรวจสอบ\" to start\n\n### Step 4: Test Detection\n\n1. **Test Gaze**: Look left, right, up, down\n2. **Test Head**: Turn head side to side\n3. **Test Objects**: Hold a phone near camera\n4. **Test Multiple Faces**: Have another person on camera\n\n## Essential Configuration\n\n### File: `config.js`\n\n```javascript\n// Most important settings:\nGAZE_THRESHOLD: 8,           // How many frames to wait\nFRAME_SKIP_OBJECT_DETECTION: 5,  // Run every 5 frames\nCOCO_SSD_CONFIDENCE: 0.50,   // Object detection accuracy\n```\n\n### Quick Adjustments\n\n**Too many false alarms?**\n```javascript\nGAZE_THRESHOLD: 12        // Increase this\nHEAD_THRESHOLD: 12\nFRAME_SKIP_OBJECT_DETECTION: 7  // Or increase this\n```\n\n**Missing real issues?**\n```javascript\nGAZE_THRESHOLD: 5         // Decrease this\nCOCO_SSD_CONFIDENCE: 0.40 // Or decrease this\n```\n\n**Slow performance?**\n```javascript\nFRAME_SKIP_OBJECT_DETECTION: 8   // Increase frame skip\nFRAME_SKIP_FACE_DETECTION: 4\n```\n\n## Common Tasks\n\n### Monitor Performance\n\nOpen browser DevTools (F12) → Performance tab:\n\n1. increase frame skipping\nFRAME_SKIP_FACE_DETECTION: 4        // was 3\nFRAME_SKIP_OBJECT_DETECTION: 7      // was 5\nCOCO_SSD_CONFIDENCE: 0.60            // was 0.50\n```\n\n## Key Features Explained\n\n### 👁️ Gaze Direction\n\n- **Green center**: Looking at screen ✓\n- **Yellow left/right/up/down**: Looking away ⚠️\n- **Red**: Looking away too long 🚨\n\n### 🔄 Head Pose\n\n- **Green center**: Natural position ✓\n- **Yellow**: Tilted >15° ⚠️\n- **Red**: Extreme tilt or turned away 🚨\n\n### 👥 Face Count\n\n- **1**: Perfect ✓\n- **0**: Nobody in frame 🚨\n- **2+**: Multiple people detected 🚨\n\n### 📱 Objects\n\nDetects:\n- 📱 Cell phone\n- 📖 Book/Paper\n- 💻 Laptop\n- 🎮 Remote/Controller\n- ⌚ Tablet\n\n## Alert Types\n\n| Alert | Severity | Meaning |\n|-------|----------|----------|\n| Looking Up | ⚠️ Yellow | Student looking at ceiling |\n| Looking Down | ⚠️ Yellow | Student looking at desk |\n| Looking Left/Right | 🚨 Red | Student looking at neighbor |\n| No Face | 🚨 Red | Student left frame |\n| Multiple Faces | 🚨 Red | Someone else in frame |\n| Phone Detected | 🚨 Red | Phone found in frame |\n| Book Detected | 🚨 Red | Notes/cheat sheet found |\n\n## Advanced Tips\n\n### Calibration for Individual\n\n1. Sit normally at desk\n2. Look at screen center for 10 seconds\n3. Observe threshold values in console\n4. Adjust based on that person's baseline\n\n### For Multiple Students\n\n```javascript\n// In config.js - use stricter thresholds\nGAZE_THRESHOLD: 12         // Takes longer to alert\nFRAME_SKIP_OBJECT_DETECTION: 6  // Fewer false positives\n```\n\n### Monitor via Network\n\n```bash\n# Share localhost on network\nnpm run dev -- --host\n\n# Then access from other computer:\n# http://YOUR_IP:5173\n```\n\n## Best Practices\n\n✅ **DO**:\n- Test camera before exam starts\n- Ensure good lighting\n- Position camera at eye level\n- Use strong WiFi connection\n- Review logs after exam\n- Keep environment quiet\n\n❌ **DON'T**:\n- Use cheap USB camera\n- Place camera too low/high\n- Use backlighting\n- Block camera view\n- Close other essential apps\n- Use during power-saving mode\n\n## Performance Targets\n\n| Target | Good | Excellent |\n|--------|------|----------|\n| FPS | 20+ | 28+ |\n| CPU | <50% | <30% |\n| Memory | <250MB | <180MB |\n| Alerts/Min | <5 | <2 |\n\n## Getting Help\n\n### Debug Mode\n\n```javascript\n// In config.js\nDEBUG: {\n  ENABLED: true,\n  LOG_FRAME_RATE: true,\n  LOG_DETECTION_RESULTS: true,\n}\n\n// Then check browser console for detailed logs\n```\n\n### Common Console Messages\n\n```\n[PERF] Main Loop: avg=12.34ms ✓ Good\n[PERF] Main Loop: avg=50.00ms ⚠️ Slow\n[ERROR] FaceMesh not loaded ✗ Problem\n```\n\n### Performance Report\n\n```javascript\n// Run in console\nperformance.memory\n// Shows: usedJSHeapSize, totalJSHeapSize\n\n// If > 300MB = memory leak suspected\n```\n\n## Next Steps\n\n1. **Read** `README.md` for full documentation\n2. **Review** `OPTIMIZATION.md` for tuning\n3. **Check** `config.js` for all settings\n4. **Deploy** with `npm run build`\n\n---\n\n**Version**: 2.0.0  \n**Last Updated**: May 2026  \n**Status**: Production Ready ✅\n"
